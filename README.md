# SnapJoy-TestTask

Проект построен на принципах:
разделения ответственности (SRP)
слабой связанности систем
компонентного подхода Unity

Ключевым элементом является интерфейс IInteractable, который задаёт единый функционал для всех интерактивных объектов. Это позволяет добавлять новые механики без изменения существующего кода.
Состояние игрока управляется через PlayerState (Free, Dialogue, Inspect), что централизует контроль над вводом и поведением.

Диалоговая система реализована через DialogueNode и DialogueChoice, что упрощает расширение и редактирование.
Система квестов вынесена в отдельный QuestManager, отвечающий за логику и UI.

Основные фичи:
Все взаимодействие через наведение камеры на интерактивный обьект и нажатие клавиши "E"
Диалоговая система с выбором и ветвлением (различные концовки и т.д.), система предметов (подбор, осмотр, описание), квестовая система, NPC и их настройка в зависимости от типа (простой диалог либо квестовый персонаж с заданиями)

Интерактивные объекты на сцене:
сундук (открытие ключом)
лампа (переключение состояния включено/выключено)
вентиль и дверь (механика удержания)
сокеты для предметов
